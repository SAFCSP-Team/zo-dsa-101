# Introduction to Data Structures and Algorithms


## Concept
A **data structure** is simply a *way of organizing, storing, and manipulating data*. This definition is what most of us read about and know, and it is correct. Data structures help us organize data and manipulate it easily. But here, instead of memorizing the definition, we will take each keyword individually and explain it to understand what it means.

## Data
Data is the **raw information** we want to work with. In programming, data comes in two forms:

- **Primitive types**: integers, doubles, booleans, characters

- **Non‑primitive (composite) types**: objects made of multiple primitives


Data structures exist because raw data on its own is difficult to manage. To see why, imagine having a large collection of toys scattered randomly across the floor. If nothing is organized, finding a specific toy becomes slow and frustrating. But once you sort the toys into boxes  (e.g, dolls in one box, cars in another, balls in a third), everything becomes easier. Now, when you want a doll, you know exactly where to look.

The same idea applies in programming. If you have a collection of unorganized integers, searching for a specific value becomes time‑consuming. But once you place those integers into a structured form (e.g, an array), it becomes much easier to locate, update, or manipulate them.

## Organize
Organizing data can be done in many different ways. Returning to the toys example from the previous section, there isn’t just one correct method to arrange them. You could sort toys into boxes by category (dolls, cars, balls), by color, or even by size.
Each method is a valid form of organization, but each one affects how quickly you can find a specific toy. Some arrangements make searching easier, while others make it harder.


### Organization and Data Structure
**Organization** is **Data Structure**. Whether we are organizing toys, plates, or books, we are choosing a specific structure(organization) for how items are stored and accessed.

In programming, the same principle applies. If we want to make accessing, searching, and handling data easier, we choose an appropriate data structure. Depending on our needs, this structure might be an array, stack, queue, linked list, tree, or any other way of arranging data.

Each structure offers different strengths, and choosing the right one is the key to writing efficient programs.

#### Organization Components
Most data structures are built from two fundamental components. How we use and combine these components determines the type of organization (data structure) we create. These components are **nodes** and **pointers**.

![image 3 (node & pointer image) ]()
#### Node
A node is the basic container that holds data. This data can be primitive (like an integer) or non‑primitive (like an object).

>[!NOTE]
>In many diagrams, a node is represented as a simple circle or box containing the data value.


#### Pointer
A pointer(aka "reference") is a link that connects one node to another.

To visualize this, imagine waiting in a long car line at a food‑truck drive‑through. You usually can’t see which car is currently being served; you only know who is directly in front of you. In other words, you can point to the car ahead, but not to the entire line.

![image 4 (car line as nodes) ]()

This car‑line behavior resembles a **linked list**, where each node points to the next. If we extend this idea and allow each node to have multiple pointers, we can form more complex structures such as **trees**.

![image 5 (tree node and pointers image) ]()

Pointers are the backbone of many data structures, and by changing how many pointers a node has and how those pointers connect nodes together, we can create entirely different organizations such as linked lists, trees, graphs, and more.

## Summary
Data structures and algorithms can be summarized into three components,
- **Data** → what we store
- **Structure** → how we organize it
- **Algorithms** → how we manipulate it
